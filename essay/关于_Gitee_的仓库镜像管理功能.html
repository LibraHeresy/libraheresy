关于 Gitee 的仓库镜像管理功能 ​

前言 ​

国内的 IT 从业者，上 Github 的时候一定被 The Great Way 折磨过，它不是一直不通，而是时通时不通，非常难受。

而对于一部分不想用梯子，或者没有梯子可用的人来说，Gitee 的仓库镜像功能可能是他比较好的选择之一。

优点 ​

  1. 目前不收费（2023.04.22）

  2. 速度很快（我试用了几次，都没有出现卡壳的情况）

  3. 提供 pull 和 push 两种操作，满足同步和更新 Github 仓库的需求

缺点 ​

  1. 一个 Gitee 仓库只能同步一个 Github 仓库，同一操作只允许新建一个

  2. 操作是直接覆盖，而不是 commit

  3. 不知道啥时候收费，建议个人使用

步骤 ​

添加镜像 ​

  1. 打开仓库的 管理 页面，选择 仓库镜像管理 菜单，点击 添加镜像

  1. 授予 Github 仓库权限以后，选择 镜像仓库，这里会获取展示你 Github 的仓库列表，之后填写私人令牌，私人令牌如何获取

更新仓库 ​

push 操作会在你往 Gitee 仓库 push 的时候，自动进行；pull 操作按照文档是支持手动和自动两种方式，自动就是 Webhook 触发，手动就是自己点更新按钮。
